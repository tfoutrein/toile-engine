## 🔴 Sécurité — à traiter (nécessite des décisions)

### S1 — [HIGH] MCP : aucune containment de `project_dir` (path traversal lecture/écriture)
`crates/toile-mcp/src/lib.rs` — `resolve()` = `project_dir.join(relative)`. `Path::join`
remplace la base sur un chemin absolu et ne normalise pas `..`. Tous les outils
(`load_scene`, `create_scene`, `create_entity`, `create_particle_emitter`,
`instantiate_prefab`…) prennent un chemin/nom contrôlé par le client IA, sans validation.
→ écriture de JSON hors du projet (`{name:"../../tmp/x"}`), lecture de tout fichier qui
*désérialise* en scène valide. (Atténué : lectures filtrées par désérialisation JSON,
écritures = JSON structuré uniquement, serveur stdio local même-utilisateur — donc
**high**, pas critical.)
**Fix** : helper de containment partagé — rejeter chemins absolus et composants `..`,
puis `canonicalize().starts_with(canonicalize(project_dir))`; forcer basename+extension;
allowlist `^[A-Za-z0-9._-]+$`.

### S2 — [HIGH] ZIP-slip + pas de garde contre les bombes de décompression
`crates/toile-asset-library/src/scanner.rs:56` — `entry.mangled_name()` (la doc `zip`
recommande `enclosed_name()`), aucune limite de nombre d'entrées / taille décompressée /
ratio, entrées symlink non filtrées. Pack ZIP importé via dialogue.
**Fix** : `enclosed_name()` + re-vérif `starts_with(target)`, caps (entrées, taille/entrée,
budget total), refuser symlinks.

### S3 — [HIGH] openssl 0.10.76 — 8 advisories RustSec/CVE (mémoire)
`reqwest 0.12` est tiré avec ses features par défaut → `default-tls` → `native-tls` →
`openssl 0.10.76` (CVE-2026-41678/41681/41898/41676/42327/44662/45784/41677, corrigés en
0.10.78–0.10.80). Sur macOS la TLS passe par Security.framework, mais le code vulnérable
est compilé/lié.
**Fix (recommandé)** : passer à rustls — `reqwest = { version="0.12", default-features=false,
features=["json","blocking","rustls-tls"] }` (rustls déjà dans l'arbre, sans advisory).
Ajouter `cargo-deny`/`cargo-audit` en CI.

### S4 — [MEDIUM] Sandbox Lua incomplète
`crates/toile-scripting/src/vm.rs` — `Lua::new()` puis nil de `os/io/loadfile/dofile`
seulement. *Nuance vérifiée* : en mode sûr mlua, `debug` et `package.loadlib` natif ne sont
**pas** atteignables (l'évaluation initiale les surestimait), mais `load`/`loadstring`,
`string.dump`, `require` pur-Lua restent ; aucune limite mémoire ni d'instructions.
**Fix** : `Lua::new_with(StdLib::TABLE|STRING|MATH, …)` + nil de `load/loadstring/string.dump/
package/require`; `set_memory_limit` + hook de comptage d'instructions. Ne jamais exécuter de
script non fiable (la chaîne d'import IA élargit la surface).

### S5 — [MEDIUM] Clé API IA stockée en clair, lisible par tous
`crates/toile-editor/src/ai/config.rs` — `~/.toile/…json` écrit avec l'umask par défaut
(~0644), pas de `set_permissions(0o600)`. (Positif : les clés ne sont jamais loguées.)
**Fix** : 0600 fichier / 0700 dossier via `PermissionsExt`, écriture temp+rename, ou keychain.

### S6 — [MEDIUM] `base_url` non validée (fuite de clé / SSRF) dans l'import IA d'assets
`crates/toile-asset-library/src/ai_import.rs` — l'URL est `format!("{}/v1/messages", base_url)`
avec la clé en en-tête ; un `base_url` hostile reçoit la clé. Corps d'erreur du provider
réinjectés bruts dans l'UI.
**Fix** : allowlist https d'hôtes connus, refuser IP-littérales/localhost sauf opt-in, ne
jamais interpoler la clé dans les messages d'erreur.

### S7 — [LOW] Scanner suit les symlinks (`follow_links(true)`)
`scanner.rs:12` — un pack avec un symlink hors-racine est parcouru/indexé.
**Fix** : `follow_links(false)` + vérifier que le chemin canonique reste sous la racine.

### S8 — [LOW] `create_tilemap` MCP : allocation non bornée
`toile-mcp/src/lib.rs` — `vec![0; (w*h) as usize]` avec w/h IA (OOM, overflow 32-bit).
**Fix** : clamp w/h/tile_size + `checked_mul`.

### S9 — [LOW] `bug_reporter` : `repo`/`component` non validés, labels `--force`
`toile-editor/src/ai/bug_reporter.rs` — bon usage d'arg-vectors (pas d'injection shell),
mais `repo` IA + `gh` avec les creds de l'utilisateur ⇒ autorité large.
**Fix** : allowlist du repo, regex `component`, confirmation utilisateur avant dépôt.

## 🟠 Concurrence

### C1 — [HIGH] Le copilote IA se bloque définitivement si on ferme le panneau pendant une requête
`crates/toile-editor/src/ai/chat_panel.rs` — `check_ai_response()` n'est appelé que quand
`editor_mode == AICopilot`. Fermer le panneau (passe en `Entity`) laisse `ai_loading=true`
pour toujours ; le garde d'envoi exige `!ai_loading` ⇒ copilote inutilisable jusqu'au
redémarrage.
**Fix** : poller `check_ai_response()` inconditionnellement dans la boucle d'update, ou
réinitialiser `ai_loading`/drop `ai_response_rx` à la fermeture.

### C2 — [HIGH] Appel réseau bloquant sur le thread UI (gel de l'éditeur), sans timeout
`crates/toile-editor/src/ai/config.rs` — `fetch_models`/`fetch_openai_models` utilisent
`reqwest::blocking` sans `.timeout(...)`, appelés dans des closures egui (perte de focus du
champ clé, bouton refresh, engrenage). Un endpoint lent/black-hole gèle l'UI indéfiniment.
**Fix** : déporter sur un thread + canal mpsc (déjà le pattern de `send_ai_message`) + timeout.

### C3 — [LOW] Divers : signal de fin de capture des logs de jeu basé sur le contenu
(`panels/overlay.rs`), `AsyncLoader` sans `Drop`/join ni backpressure (exemple seulement),
`ScriptWatcher` = plomberie morte non câblée (canal non drainé si jamais utilisé).

## 🟡 Correction / robustesse

### R1 — [MEDIUM] Lecteur binaire Aseprite sans bornes → crash éditeur sur `.ase` malformé
`crates/toile-assets/src/aseprite.rs:47-89` — `byte/word/dword/bytes/string` indexent
`self.data[self.pos..]` sans vérifier la longueur ; `chunk_data_end - r.pos` peut
sous-déborder. `parse_ase` renvoie `Result` mais **panique** en interne. Atteignable depuis
le navigateur d'assets / l'éditeur de sprites (clic sur un `.ase` d'un pack téléchargé).
*(Le clamp de tags F4 est posé, mais les primitives `BinReader` restent à rendre faillibles.)*
**Fix** : primitives renvoyant `Result`/`Option` (`checked_add` + vérif longueur),
arithmétique `checked/saturating`, test de fuzz sur entrées tronquées.

### R2 — [LOW] Loaders TTF/Aseprite/LDtk paniquent au lieu de renvoyer `Result`
`ttf.rs:21`, `animation.rs:92/149`, `ldtk.rs:183/185`. Seul `load_ttf` est réellement sur un
chemin de prod (police de projet présente mais corrompue → crash du runner). Les variantes
LDtk/Aseprite paniquantes n'ont pas d'appelant de prod.
**Fix** : renvoyer `Result` et logguer+ignorer, comme `load_ldtk_scenes`/`load_ase_file` le font déjà.

### R3 — [LOW] `sine` divise par `period` ⇒ NaN si `period == 0`
`toile-behaviors/src/sine.rs:56`. L'inspecteur clampe (0.1..60), mais JSON brut / outil IA
`add_behavior` ne clampe pas. **Fix** : `let period = config.period.abs().max(1e-3);`.

### R4 — [LOW] Clés de timer d'événements non uniques (`idx*1000`)
`toile-events/src/executor.rs` — collision si ≥1001 events frères ou imbrication profonde
(overflow `usize`). **Fix** : ids stables par événement à la charge.

*(Écartés après vérif : « EveryNSeconds rate des events en cas de lag » — faux, le timestep
est fixe ; « load_ldtk paniquant » — fonction morte, la variante `Result` est utilisée.)*

## 🔵 Cohérence / architecture

### X1 — [HIGH] Prefabs : `behaviors`/`event_sheet` legacy écrits mais jamais lus → perte de données
`crates/toile-scene/src/prefab.rs` — `instantiate()` ne lit que `self.entity` ; les champs
top-level « kept for backward compat » sont ignorés. **Or** `toile new <template>`
(`toile-cli/src/templates.rs`) génère justement des prefabs dont les behaviors ne sont **que**
top-level → à l'instanciation (runner & MCP) les entités spawnent **sans behaviors**.
**Fix** : à la charge, migrer top-level → `entity` si vide ; et corriger le générateur CLI
pour peupler `entity.behaviors` (+ `#[serde(skip_serializing)]` sur les champs legacy).

### X2 — [MEDIUM] Actions d'event `MoveToward` / `PlayAnimation` émises mais ignorées
`toile-runner/src/game_runner.rs:644` — `_ => {} // not yet implemented`. Définies, parsées,
émises… puis no-op silencieux (feature event-sheet v0.5 « supportée » de bout en bout mais
inerte). **Fix** : implémenter les deux bras, ou `log::warn!` + le signaler dans l'éditeur.

### X3 — [MEDIUM→low] `toile-core` dépend de `wgpu` (viole ADR-009 « core ne dépend de rien »)
Pour un seul `impl From<Color> for wgpu::Color` (9 lignes). Impose le plus gros arbre de deps
au crate fondamental. **Fix** : déplacer l'impl dans `toile-graphics`, retirer `wgpu` de
`toile-core/Cargo.toml`.

### X4 — [LOW] ADR-009 documente des traits backend inexistants
`trait Platform`/`GraphicsBackend`/`AudioBackend` n'existent pas (structs concrètes ; winit en
dur, pas de SDL3). **Fix** : mettre l'ADR à jour ou introduire les traits.

### X5 — [LOW] Déclaration `toile-core` incohérente (path+features vs workspace) sur 8 crates.
**Fix** : `toile-core = { workspace = true, features = ["serde"] }` partout.

### X6 — [INFO] `SceneData::default()` ⇒ `next_id=0` (vs `new()`=1), footgun latent d'ID.

## ⚡ Performance (audit + recherche bonnes pratiques)

| # | Sév. | Sujet | Fichier |

| P1 | high | **Pas d'interpolation de rendu** : `alpha` (Fix Your Timestep) calculé puis jeté → micro-stutter si refresh ≠ update_hz | `toile-app/src/lib.rs:612` + `toile-core/src/time.rs` |

## Dépendances — synthèse

- **Action conseillée** : openssl → rustls (S3) ; ajouter `cargo-deny` en CI.

  (0.9 → 0.12), `egui` (0.33 → 0.34), `rapier2d` (0.32 → 0.33).
